# مِعيار

منصة تحليل الالتزام بكود المنصات، تساعد على مطابقة المواقع مع المعايير وتوثيق نتائج عدم المطابقة بأدلة فنية قابلة للمراجعة.

## ماذا تفعل النسخة الحالية؟

- تفتح الرابط المدخل باستخدام Playwright.
- تفحص الصفحة وصفحات داخلية مهمة بحد أقصى 3 صفحات.
- تطبق مجموعة معايير مبنية على ملف كود المنصات وقواعد Rules Catalog.
- تحسب نسبة تطابق حقيقية مبدئية من نتائج الفحص الآلي.
- تعرض درجة لكل معيار: مطابق، غير مطابق، يحتاج مراجعة.
- تعرض المخالفة مع دليلها: selector، HTML، ولقطة صورة للعنصر المخالف عند توفرها.
- تستخدم axe-core لفحص الوصولية.
- لا تعرض أي نتائج تجريبية. أي فحص جديد يعتمد على realScan أو يظهر فشل الفحص بوضوح.

## التشغيل

```bash
npm install
cd server
npx playwright install chromium
cd ..
npm run dev
```

الواجهة:

```text
http://localhost:5173
```

الخادم:

```text
http://localhost:4000
```

## سياسة النتائج

- لا توجد نتائج Mock أو demo في الواجهة.
- أي فحص جديد من `/scan` ينتج تقريرًا حقيقيًا مبدئيًا مبنيًا على الموقع المدخل.
- إذا فشل الفحص الحقيقي، تظهر رسالة فشل واضحة بدون استخدام أرقام تجريبية.

## كيف تُحسب النسبة؟

النسبة تُحسب من نتائج المعايير في `criteriaResults`:

- المعيار المطابق يأخذ 100.
- المعيار غير المطابق يأخذ 0.
- المعيار الذي يحتاج مراجعة يدوية يأخذ 50 ويظهر بوضوح كـ "يحتاج مراجعة".
- كل معيار له وزن، والنسبة النهائية هي متوسط وزني.

## الملفات المهمة

- `server/src/services/realScanner.service.js`: فتح الموقع، تحليل DOM، axe-core، والتقاط صور المخالفات.
- `server/src/services/scan.service.js`: بناء التقرير الحقيقي، حساب النسبة، درجات المعايير، النتائج والملخص.
- `server/src/data/rulesCatalog.js`: كتالوج المعايير.
- `client/src/pages/ResultsPage.jsx`: عرض النتائج، درجات المعايير، صور المخالفات، وجداول الصفحات والعناصر.
- `client/src/pages/IssuesPage.jsx`: عرض مخالفات الفحص الحالي عند تمرير scanId.

## ملاحظات

هذه نسخة تحليل آلي داعمة وليست اعتمادًا رسميًا نهائيًا. المعايير التي لا يمكن فحصها آليًا بالكامل تظهر بحالة "يحتاج مراجعة" بدل إعطاء حكم غير دقيق.


## تحديث الثبات والصور
- لا تُستخدم أي بيانات تجريبية في الفحص الجديد.
- نسبة التطابق العامة تُحسب من المعايير الآلية القابلة للقياس فقط، بينما تظهر معايير "يحتاج مراجعة" بشكل منفصل ولا تخفض النسبة.
- روابط صور المخالفات تعمل عبر خادم Express على /screenshots مع proxy من Vite.

## ملاحظة على حساب نسبة التطابق
تعتمد النسبة العامة على المعايير الآلية القابلة للفحص فقط. المعايير التي تحتاج مراجعة بشرية أو بصرية تظهر في التقرير كـ "يحتاج مراجعة" ولا تُخفض نسبة التطابق. كما يتم احتساب المخالفات المتكررة حسب نوع المعيار والخطورة حتى لا تنخفض الدرجة بسبب تكرار نفس الملاحظة في عدة عناصر أو صفحات.


## تحديثات الدقة الأخيرة
- أزيل فحص lazy loading من المخالفات والمعايير لأنه ليس معيارًا مباشرًا في ملف كود المنصات المستخدم.
- الفحص الشامل يفحص حتى 6 صفحات من نفس الموقع بدل 3 صفحات، مع تقليل العدد في الفحص السريع لتجنب البطء.
- صور المخالفات لا تستخدم لقطة شاشة عامة كبديل؛ إذا تعذر التقاط صورة العنصر، تظهر المخالفة بالدليل النصي والـ selector بدون صورة مضللة.
